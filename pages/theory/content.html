Html & CSS: BEM naming

  • Naming convention for classes
  • Block Element Modifier (BEM)
  • Verbose

Usage

  • block__element--modifier
  • block--modifier
  • element--modifier

Modifier means appearance or behaviour

Example


 <div class="dialog dialog--hidden">
    <p class="dialog__msg"></p>
    <button class="dialog__button-yes"></button>
    <button class="dialog__button-no"></button>
 </div>

Services: REST

What stands for?

Representational State Transfer

  • Network of webpages (state-machine).
  • The user progresses through an application by selecting links (state transitions).
  • Resulting in pages (states) being transferred to the user and rendered for their use.

What is?

  • Architectural Style, Not a Standard.
  • Describes how a well-designed Web application behaves.
  • Consistency by following web standards and conventions.

Characteristics:

Client-Server
A pull-based interaction style: consuming components pull representations.
Stateless
Each request must contain all the information necessary to understand the request.
Request cannot take advantage of any stored context on the server.
Cache
Use of cache to improve network efficiency.
responses must be capable of being labeled as cacheable or non-cacheable.
Uniform interface
CRUD operations (Create, Read, Update, Delete).
All resources are accessed with a generic interface (HTTP: GET, POST, PUT, DELETE).
Named resources
The system is comprised of resources which are named using a URL.
Interconnected resource representations
Hypermedia as the Engine of Application State (HATEOAS)
Resources are interconnected using URLs, enabling a client to navigate between states.
Layered components
Middlewares can be inserted between clients and resources to support performance, security, etc.

Principles:

  • Use of JSON, SSL/TLS, and HTTP status codes.
  • Handle errors with standard error codes.
  • Loose coupling between Client and Server.
  • Use of nouns only since the HTTP methods indicate the action we want to take.
  • Paths of nested resources should come after the path of the parent resource.

HTTP Statuses

200 OK Everything is fine
400 Bad Request Client-side input fails validation
401 Unauthorized User isn’t authenticated
403 Forbidden User is authenticated, but it’s not allowed to access
404 Not Found Resource is not found
500 Internal server error Generic server error
502 Bad Gateway Invalid response from an upstream server
503 Service Unavailable Something unexpected happened on server side

APIs

Usually to fetch data from an API endpoint you need:

  • Account
  • API key

The API key usually have to be included in the request URL.


http://api.openweathermap.org/data/2.5/weather?q=London&APPID=1111111111

Most API services offers a paid tier.

Any data in client-side is vulnerable.
Make the request server-side.
